Objetivo de la adquisición: fortalecer la infraestructura del Data Center institucional mediante servidores de alto rendimiento que incrementen capacidad de procesamiento, memoria y almacenamiento, optimicen el desempeño de las aplicaciones institucionales y permitan escalabilidad futura.

Dos datos que cambian el análisis

La plataforma Nutanix adquirida hace un año está dedicada en exclusiva al Proyecto Tránsito de la Policía Nacional — sus recursos no están disponibles para esta consolidación, así que esta decisión arranca de cero, no es una ampliación de lo que ya existe. Además, el presupuesto institucional ya no alcanzó para renovar las licencias VMware vSphere/vCenter tras el cambio de modelo de licenciamiento de Broadcom, lo que dejó la plataforma de virtualización actual sin soporte. Cualquier arquitectura nueva hereda ese mismo riesgo si vuelve a depender de un solo proveedor para fijar el precio del software año a año.

Cómo se ve cada arquitectura

La pregunta de fondo es dónde vive el almacenamiento y cuántos proveedores quedan dentro de esa decisión.

VMware + IBM Storage

Cómputo virtualizado con VMware (sin vSAN) y almacenamiento dedicado IBM Storage son capas separadas. Cada capa se renueva, escala y se reemplaza de forma independiente.

Lenovo + VMware vSAN

Cada nodo trae cómputo y almacenamiento integrados de fábrica con VMware vSAN como un solo producto. No se aceptan nodos compatibles ni "Ready Nodes" de otros ensambladores.

Ambas arquitecturas deben cumplir la misma exigencia funcional: réplica entre sitio principal y alterno para continuidad operativa.

Comparativa económica

A esta escala, ambas opciones usan VMware como hipervisor — el costo de licenciamiento por core de Broadcom aplica igual en las dos. La diferencia económica real está en otro lugar.

!

El presupuesto de la institución ya no alcanzó para renovar vSphere/vCenter bajo el modelo de licenciamiento de Broadcom. Sea cual sea la arquitectura elegida, ese costo de licenciamiento por core hay que cuantificarlo con cifras reales antes de comprometer un número con la entidad — en convergente y en hiperconvergente por igual.

Comparativa operativa

DNTIC ya está construyendo un equipo de seguridad y operación más maduro — WAF, XDR y PAM forman parte de esta misma adquisición. Esa madurez técnica es justamente lo que permite aprovechar mejor una arquitectura con más capas que administrar.

Consumo multi-protocoloIBM Storage expone bloques, NFS, SMB y S3 de forma nativa para distintos consumidores (video wall, evidencias, bases de datos)También lo ofrece, pero atado al stack de software del fabricante de HCI

Ventaja real de convergente aquí

  • Cada capa se puede reemplazar o renovar sin tocar las demás
  • Más proveedores pueden cotizar cómputo y storage por separado
  • El equipo técnico que ya se está formando con WAF/XDR/PAM puede operar storage a nivel granular

Ventaja real de hiperconvergente aquí

  • Una sola consola para cómputo, storage y virtualización
  • Despliegue inicial más simple, menos piezas que integrar
  • Un solo proveedor para escalar la totalidad del stack

Riesgo y continuidad

Esta es la pregunta que más le sirve a la entidad: ¿cuántos proveedores distintos sostienen la arquitectura nueva?

VMware + IBM Storage — capas independientes3 proveedores distintos
Lenovo + VMware vSAN — sistema integrado1 proveedor para todo
!

La plataforma de virtualización actual quedó sin soporte porque dependía de un solo proveedor que cambió las reglas de licenciamiento de un año a otro. Un "sistema integrado" hiperconvergente reproduce exactamente ese mismo patrón de dependencia — hardware, software y soporte de un único fabricante — con la diferencia de que ahora también incluiría el almacenamiento dentro de ese mismo punto único de control.

Recomendación

"La hiperconvergencia es más moderna, por eso conviene."

La pregunta correcta es cuál arquitectura evita repetir la dependencia de proveedor único que dejó sin soporte la plataforma actual — no cuál tecnología suena más nueva.

Conclusión

Para este entorno, una arquitectura convergente con VMware y almacenamiento dedicado IBM Storage preserva la capacidad de cotizar y renovar cada capa por separado, evitando que toda la plataforma quede otra vez sujeta a las decisiones comerciales de un solo fabricante. Cumple los mismos requisitos funcionales de réplica y continuidad entre sitios que exige el proyecto, sin concentrar cómputo, almacenamiento y software en un único proveedor.
